# Conspiracy — UI & Menus Design Document

*"Clarity through irony."*

**Game:** Conspiracy: A Simulation of Power, Paranoia, and Poor Decision-Making
**Author:** Alexei Fedotov

---

> **Architecture note:** The client is a browser-based viewer and order collector (GitHub Pages). It has no simulation state of its own. All game logic lives in the Python engine. Every UI panel reads from world-state JSON; every player action ultimately submits an orders PR. UI design must reflect this: panels display, Orders panel acts.

# 2. MAIN GAME CLIENT

> The game client is a single-page layout with persistent panels. No screen transitions mid-session — panels expand, collapse, or slide over the map.

## 2.1 Top Bar (Persistent)

- **Dominion Name**
- **Turn Counter** + **Next Turn ETA**
- **Resources Strip**
  - Trust (primary currency — generated by public promises, destroyed by fulfilling them)
  - Influence (soft power across factions)
  - Belief (fuels economy and magic; collapses without ideological maintenance)
  - Entropy (accumulated systemic instability; high entropy triggers cascade events)
  - Paperwork (bureaucratic backlog; zero = efficient; overflow = collapse)
- **Alert Bell** — urgent events awaiting response
- **Orders Status** — "Draft", "Submitted", "Processing", "Resolved"
- **Session Menu** (≡)

// Trust and Belief are the two most important metrics. Display them prominently with trend arrows.
// Orders Status reflects the current PR state in the GitHub CI pipeline.

## 2.2 World Map (Central Canvas)

### Map Modes (toggle strip)

- Administrative *(regions, dominion boundaries, control colour)*
- Trust Flow *(where Trust is accumulating or draining)*
- Belief Density *(ideological saturation by region)*
- Entropy Heat Map *(instability — red = imminent event)*
- Faction Influence *(factional penetration overlaid on regions)*
- Infrastructure *(roads, faith networks, industrial nodes)*
- Hero Positions *(active agents and their current missions)*

### Map Interactions

- Click region → opens Region Panel (§3)
- Click hero marker → opens Hero Sheet (§5.1)
- Right-click → context menu: *Assign Mission / Observe / Fabricate Report*
- Scroll to zoom; drag to pan

## 2.3 Side Panel (Right, Collapsible)

Tabs:

- **Notifications** — event feed, faction messages, divine complaints
- **Dominion** — aggregate stats and Control Layer overview
- **Orders** — current turn's order queue (§6)
- **Chronicle** — Event Viewer / history log (§7)
- **Statistics** — Chart.js graphs of metrics over time (§8)

# 6. ORDERS PANEL

> The Orders panel is the core gameplay interface. All player actions are assembled here and submitted as a turn PR. Orders are validated client-side before submission; the Python engine validates and resolves them server-side.

## 6.1 Order Queue

- List of staged orders for the current turn
  - Order Type
  - Target
  - Assigned Hero / Resource
  - Estimated cost (Trust, Influence, Paperwork)
  - Conflict warnings *(e.g., two heroes ordered to the same region)*
- Reorder / Remove orders
- **Trust Budget Summary** *(remaining vs. committed)*
- **Validate Orders** *(client-side pre-check)*

## 6.2 Order Categories

- **Diplomacy**
  - Propose Treaty (Trade, Alliance, Non-Aggression, Mutual Audit Pact)
  - Break Treaty *(generates Entropy spike in affected region)*
  - Offer Tribute
  - Declare Ideological War
  - Demand Compliance *(ultimatum with turn deadline)*
- **Economy**
  - Adjust Budget Allocation
  - Open / Close Trade Route
  - Trigger Belief Event *(spend Belief to manufacture economic miracle)*
  - Sponsor Industry
- **Military**
  - Move Garrison
  - Fortify Region
  - Launch Campaign *(assigns multiple heroes + garrisons)*
  - Withdraw
- **Administrative**
  - Issue Decree *(policy with ongoing Trust cost)*
  - Repeal Decree
  - Appoint / Remove Advisor
  - Assign Governor
  - File Appeal *(contest an NPC faction's action through the Bureau)*
- **Heroic**
  - Assign Mission (§5.3)
  - Recall Agent
  - Promote Hero
- **Infrastructure**
  - Queue Construction
  - Cancel Construction
  - Demolish *(rare; releases Entropy)*
- **Ideology**
  - Fund Propaganda Campaign
  - Suppress Ideology
  - Found Cult *(high cost; high Belief return; high risk)*
  - Register Conspiracy with the Bureau *(legitimises a scheme; reduces risk, reduces deniability)*

## 6.3 Submit Turn

- **Review Orders** (summary list)
- **Estimated Turn Outcome** *(heuristic preview — not guaranteed)*
- **Submit Orders** → opens PR via GitHub API
- **Order Status Tracker** → shows CI pipeline progress (Queued → Validating → Resolving → Committed)
- **Cancel Submission** *(only before CI picks up the PR)*

# 7. EVENT VIEWER / CHRONICLE

> Fetches and renders `history/events.log` after each turn resolves. This is both the post-turn debrief and the long-form narrative record.

## 7.1 Latest Turn Report

- Turn header (turn number, date, world headline)
- Events grouped by type:
  - **World Events** (global crises, god bankruptcies, ideological wars)
  - **Faction Events** (what other factions did)
  - **Dominion Events** (what happened in your regions)
  - **Hero Events** (mission outcomes, trait mutations, paranoia episodes)
  - **Diplomatic Events** (treaty responses, ultimatum resolutions)
- Each event: narrative tone, in-world language, colour-coded by severity

## 7.2 Chronicle Browser

- Searchable history of all past events
- Filter by: Turn / Faction / Hero / Region / Event Type
- Turn headers for navigation
- **Lore Mode** *(render events as fictional historical prose)*

## 7.3 Bureaucratic Inbox

- Pending decisions requiring player response before next turn
  - Treaty offers
  - Faction ultimatums
  - Bureau of Conspiracies rulings
  - Divine complaints (gods file grievances procedurally)
  - Hero loyalty crises
- Each item: Accept / Deny / Refer to Committee *(delay by 1 turn)*

# 16. VICTORY / DEFEAT CONDITIONS

> *Victory in Conspiracy is impossible; influence decays, trust evaporates, and heroes forget what they were saving. The true win condition is to understand the system before it forgets you exist.*

## 16.1 Score Conditions (Optional)

- **Ideological Dominance** — hold >60% Belief share across all regions for 10 consecutive turns
- **Bureaucratic Supremacy** — register 50 Official Conspiracies simultaneously
- **Trust Monopoly** — control >80% of global Trust for 5 turns
- **The Long Game** — survive 200 turns without Dominion collapse

## 16.2 Defeat Conditions

- Entropy exceeds threshold for 3 consecutive turns (Dominion Collapse)
- Trust reaches zero (Dominion Forgotten — ceases to exist in the simulation's records)
- All heroes defect, die, or file for independent incorporation
- The Shadow Guilds reveal the player as the Final Conspiracy (meta-ending)

## 16.3 Endings Screen

- Procedurally generated historical summary of the Dominion's arc
- Cause of collapse (or victory) in bureaucratic prose
- Unlocked lore entries
- Option: Archive Dominion as Historical Record / Restart
